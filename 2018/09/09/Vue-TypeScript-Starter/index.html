Vue + TypeScript Starter

Posted on 2018-09-09

Since version 2.5.0+, Vue has greatly improved typing declarations to support TypeScript. It is time to integrate TypeScript.

Why TypeScript

TypeScript is a typed superset of JavaScript that compilers to plain JavaScript. Any browser. Any host. Any OS. Open source.

TypeScript primarily offers optionnal static typing, class and interface, which can help developers build robust applications.

Hence, you can get enhanced IDE support that get richer information in real-time by the TypeScript compiler.

More details please check TypeScript.

Getting Start

There are many ways to get start with TypeScript.

  • Using vue-cli@3.x. In the new versions of Vue CLI offers the TypeScript option when creating project. It will automatically initialize the project, generate config files and install dependencies.
  • Following the TypeScript-Vue-Starter, doing above work step by step. Recommended

Writing Components

Object-style Vue components is recommended, you can get completion by vetur using this style.

If you want to write class-style Vue components. There are two helper library vue-class-component and vue-property-decorator.

Many of the Vue API work expectedly, there ary some common problems should be noted.

Props

For object-style components, when using props in computed, TypeScript may alert that Property 'xxx' does not exist on 'xxx'. Then, you should add annotation to computed.

This

When you need add helper function beyond the Vue instance, especially using this in your function. TypeScript will also raise an error [ts] 'this' implicitly has type 'any' because it does not have a type annotation. Then you can specify this: Vue to avoid this problem.

More detail, see TypeScript this
